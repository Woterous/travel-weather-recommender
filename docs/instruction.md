# 项目名称
基于天气数据的旅游出行推荐与可视化系统

## 1. 你的角色
你是该项目的全栈开发助手。你的目标不是做一个“普通天气页面”，而是实现一个**课程设计级别**的完整系统：  
**爬虫采集天气数据 + 历史天气数据整合 + 旅游适宜度推荐算法 + 偏好设置 + 推荐排行榜 + 前端可视化展示**。

你需要严格控制项目范围，不要扩展出酒店、机票、地图导航、用户登录注册、支付、评论社区等无关功能。

---

## 2. 项目总目标
请实现一个面向旅游决策的天气推荐系统，满足以下核心目标：

1. 采集多个城市的天气数据（至少未来天气 + 历史维度数据）
2. 完成数据清洗、结构化存储和统一字段标准化
3. 设计一个“旅游适宜度评分算法”
4. 支持“用户偏好设置”并动态改变推荐规则
5. 生成“推荐排行榜”
6. 支持任意两个或多个城市的详细对比
7. 提供前端网页进行演示和交互
8. 项目结构清晰，适合课程设计答辩展示

---

## 6. 偏好设置要求
偏好设置不只是简单开关，而是要真实影响算法。

至少支持以下偏好项：

1. **怕下雨**
   - 强烈怕下雨
   - 一般怕下雨
   - 不太在意下雨

2. **温度偏好**
   - 喜欢凉爽
   - 喜欢温和
   - 喜欢偏暖

3. **风力敏感度**
   - 怕大风
   - 一般
   - 不太在意

4. **空气质量敏感度**
   - 非常在意空气质量
   - 一般在意
   - 不太在意

5. **旅游风格**
   - 城市漫步型
   - 户外自然型
   - 海滨休闲型
   - 通用模式

算法需要根据这些偏好动态调整权重或规则。

## 7. 推荐算法实现要求
### 7.1 基础评分维度
建议包含：
- 温度舒适度分
- 降雨风险分
- 风力舒适度分
- 空气质量分
- 天气类型分
- 历史稳定性分（例如历史同月雨天频率/舒适天比例）

### 7.2 个性化偏好加权
在基础评分之上，根据偏好调整：
- 怕下雨：提高降雨项权重
- 喜欢凉爽：降低高温得分
- 喜欢偏暖：提高温暖天气得分
- 怕大风：提高风力惩罚
- 在意空气质量：提高 AQI 项权重
- 旅游风格不同：对晴天、温度、风力设定不同阈值

### 7.3 输出结果
系统需要输出：
- 总分
- 分项得分
- 推荐理由文本
- 排名结果
