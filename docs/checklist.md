# 项目验收清单

基于天气数据的旅游出行推荐与可视化系统

> 使用方式：每完成一项就在对应方框中打勾。
> 只有“硬性通过项”全部完成，项目才视为达到 V1 可验收状态。

---

## A. 已冻结基线

以下基线已由项目负责人确认：

- [x] 首版不引入 Spark、Hive、Hadoop
- [x] 技术栈采用 Python + Flask + pandas
- [x] 数据获取以公开网页抓取为主
- [x] 公开 API 仅作为补充
- [x] 样例数据仅用于开发调试
- [x] 固定城市为北京、上海、杭州、南京、成都、重庆、厦门、青岛、西安、三亚
- [x] 历史数据首版以月度统计为主
- [x] AQI 设为增强项，允许无 AQI 降级模式
- [x] 首页与排行榜页合并
- [x] 主存储固定为 SQLite，原始抓取允许 JSON 或 CSV 备份

---

## B. 范围合规

### 硬性通过项

- [ ] 项目主题仍然是“旅游天气推荐”，不是普通天气播报
- [ ] 系统围绕“采集、清洗、评分、展示”完整链路展开
- [ ] 未扩展到用户系统、地图导航、酒店机票、支付评论等无关功能
- [ ] 推荐算法仍然是规则型、可解释算法
- [ ] 历史分析仍然处于课程设计可讲清楚的范围

### 红线项

以下项目必须全部保持“未出现”状态：

- [ ] 未在未更新文档的前提下临时加入登录、地图、预订、聊天等功能
- [ ] 未在未更新文档的前提下临时更换为 Vue、React、MySQL、Spark 等大栈
- [ ] 未在未更新文档的前提下把固定城市改成任意城市
